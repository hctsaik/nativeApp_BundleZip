新增 CV 模組指南

CIM CV Framework — 工程師版 · 2026-05

CIM Platform 採用 plugin.yaml 自動掃描 機制。 只需在 scripts/ 資料夾建立子目錄並放入 plugin.yaml 與三個 Python 檔案， 重啟 sidecar 後新模組即自動出現在選單中。無需修改 engine.py。

✅
自 2026-05 起，engine.py 在啟動時自動掃描所有 scripts/*/plugin.yaml。 新模組只需建立資料夾與 yaml，不需要手動 seed 或修改任何共用檔案。

開發清單

新增一個模組共需完成以下步驟：

  1. 確認 module ID 不重複（執行 ls sidecar/python-engine/scripts/）
  2. 建立資料夾 scripts/module_{NNN}/
  3. 撰寫 plugin.yaml（必填，engine.py 自動讀取）
  4. 撰寫 __init__.py
  5. 撰寫 {NNN}_input.py — render_input() → dict
  6. 撰寫 {NNN}_process.py — execute_logic(params) → dict（禁止 import streamlit）
  7. 撰寫 {NNN}_output.py — render_output(result) → None
  8. 撰寫 {NNN}_process_test.py — 至少 8 個 pytest 測試
  9. 執行 pytest scripts/module_{NNN}/ 全部通過
  10. 重啟 sidecar，確認模組出現在選單

三層契約

每個模組由三個 Python 檔案組成，框架按序呼叫：

scripts/module_NNN/
├── __init__.py # MODULE_NAME = "模組名稱"
├── plugin.yaml # 模組唯一真相來源
├── {NNN}_input.py # render_input() → dict
├── {NNN}_process.py # execute_logic(params) → dict
├── {NNN}_output.py # render_output(result) → None
└── {NNN}_process_test.py # pytest，至少 8 個測試

執行流程

render_input() → 使用者按 ▶ 執行 → execute_logic(params) → render_output(result)

層檔案必須實作限制
Input {NNN}_input.py MUST render_input() → dict —
Process {NNN}_process.py MUST execute_logic(params: dict) → dict 禁止 import streamlit
Output {NNN}_output.py MUST render_output(result: dict) → None —
⚠️
execute_logic() 的回傳值會透過 JSON 序列化後傳給 render_output()。 只能回傳 JSON-safe 型別（見下節）。

plugin.yaml

plugin.yaml 是模組的唯一設定來源。engine.py 在啟動時掃描並自動寫入 DB，不需要手動 seed。

plugin.yaml# 必填欄位
id: module_014                 # 唯一識別碼，格式 module_NNN
vendor: cimcore                # 核心模組用 cimcore；外部貢獻者用 partner_{github_id}

JSON 序列化規則（最重要）

execute_logic() 的回傳值會序列化為 JSON，再由 render_output() 讀取。 只能回傳以下型別：

類型狀態處理方式
str, int, float, bool, None允許直接回傳
list, dict允許直接回傳（元素亦須符合規則）
bytes禁止轉 base64.b64encode(data).decode("ascii")
numpy.ndarray禁止arr.tolist() 或 base64
datetime禁止dt.isoformat()
tuple注意序列化後變 list，render_output 需轉回 tuple
# ✅ 正確：bytes 轉 base64
import base64, io
from PIL import Image

def execute_logic(params: dict) -> dict:
    img: Image.Image = ...          # 處理後的圖片
    buf = io.BytesIO()
    img.save(buf, format="PNG")
    return {
        "image_b64": base64.b64encode(buf.getvalue()).decode("ascii"),
        "width": img.width,
        "height": img.height,
    }

# ❌ 錯誤：直接回傳 bytes 或 ndarray
return {"image_bytes": buf.getvalue()}   # bytes → JSON 序列化失敗
return {"array": np_array}               # ndarray → JSON 序列化失敗

撰寫 Input 層

render_input() 負責渲染 Streamlit UI 並收集使用者輸入，最後 return dict。

{NNN}_input.pyimport streamlit as st

def render_input() -> dict:
    st.subheader("參數設定")

    # 文字輸入
    threshold = st.slider("閾值", 0, 255, 128)

    # 影像上傳（重要：只傳 bytes，不傳 Image 物件）
    uploaded = st.file_uploader("上傳影像", type=["jpg", "jpeg", "png"])
    file_bytes = uploaded.read() if uploaded else None
    image_name = uploaded.name   if uploaded else ""

    return {
        "threshold": threshold,
        "file_bytes": file_bytes,
        "image_name": image_name,
    }

撰寫 Process 層

execute_logic() 是純運算函式。嚴禁 import streamlit， 因為此層在 sidecar 的獨立 process 中執行，且需要被 pytest 測試。 

{NNN}_process.pyimport base64
import io
from PIL import Image

# ❌ 絕對不能在這裡 import streamlit

def execute_logic(params: dict) -> dict:
    file_bytes: bytes | None = params.get("file_bytes")
    threshold: int = params.get("threshold", 128)

    if not file_bytes:
        return {"error": "no_image"}

    img = Image.open(io.BytesIO(file_bytes)).convert("L")
    result = img.point(lambda p: 255 if p > threshold else 0)

    buf = io.BytesIO()
    result.save(buf, format="PNG")

    return {
        "image_b64": base64.b64encode(buf.getvalue()).decode("ascii"),
        "image_name": params.get("image_name", "output.png"),
        "threshold_used": threshold,
    }

撰寫 Output 層

{NNN}_output.pyimport base64
import streamlit as st

def render_output(result: dict) -> None:
    if result.get("error"):
        st.error(f"錯誤：{result['error']}")
        return

    st.subheader("處理結果")

    image_b64: str = result.get("image_b64", "")
    if image_b64:
        image_bytes = base64.b64decode(image_b64)
        st.image(image_bytes, caption=result.get("image_name", ""))

    st.metric("使用閾值", result.get("threshold_used", "—"))

影像在表格中的呈現

不要把 st.image 放在 table cell 裡。改用 @st.dialog：

@st.dialog("影像預覽", width="large")
def _show_preview(rec: dict) -> None:
    if rec.get("image_b64"):
        st.image(base64.b64decode(rec["image_b64"]))

def render_output(result: dict) -> None:
    rows = result.get("records", [])
    for rec in rows:
        cols = st.columns([3, 1])
        cols[0].write(rec["name"])
        if cols[1].button("預覽", key=f"view_{rec['id']}"):
            _show_preview(rec)

SQLite 持久儲存

⚠️
DB 路徑必須在呼叫時解析，不能在 import 時解析。 否則 pytest 的 monkeypatch 無法覆蓋 CIM_LOG_DIR。 
{NNN}_process.py — SQLite 樣板import os
import sqlite3
from pathlib import Path

# ❌ 錯誤：import 時解析（monkeypatch 失效）
# DB_PATH = Path(os.environ.get("CIM_LOG_DIR", "/tmp")) / "records.sqlite"

# ✅ 正確：函式內解析
def _db_path() -> Path:
    return Path(os.environ.get("CIM_LOG_DIR", "/tmp")) / "my_records.sqlite"

def _ensure_db(conn: sqlite3.Connection) -> None:
    conn.execute("""
        CREATE TABLE IF NOT EXISTS my_records (
            id    INTEGER PRIMARY KEY AUTOINCREMENT,
            name  TEXT NOT NULL,
            score REAL,
            ts    TEXT
        )
    """)
    # 安全遷移：用 PRAGMA 而非直接 ALTER（欄位已存在不報錯）
    existing = {r[1] for r in conn.execute("PRAGMA table_info(my_records)").fetchall()}
    if "new_column" not in existing:
        conn.execute("ALTER TABLE my_records ADD COLUMN new_column TEXT")

def execute_logic(params: dict) -> dict:
    with sqlite3.connect(_db_path()) as conn:
        _ensure_db(conn)
        conn.execute(
            "INSERT INTO my_records (name, score, ts) VALUES (?, ?, ?)",
            (params["name"], params["score"], params["ts"])
        )
        conn.commit()
    return {"saved": True}

跨模組導航（SWITCH_TAB）

若模組執行完需要跳轉到另一個模組，使用 postMessage 通知 Portal：

{NNN}_output.py — 跨模組導航import json
import streamlit as st
import streamlit.components.v1 as components

def _post_message(msg_type: str, payload: dict) -> None:
    blob = json.dumps({
        "type": msg_type,
        "source": "cim-platform",  # ← 必填，缺少此欄位訊息會被 Portal 靜默丟棄
        "payload": payload,
        "_cim": True,
    })
    components.html(f"<script>window.top.postMessage({blob}, '*');</script>", height=0)

def render_output(result: dict) -> None:
    st.success("處理完成")

    if st.button("📁 前往 Update →", type="primary"):
        _post_message("SWITCH_TAB", {
            "plugin_id": "module_013",
            "tab": "input",
        })
🚫
source: "cim-platform" 為必填。 Portal 的 isProtocolMessage() 會驗證此欄位，缺少則訊息靜默丟棄。 此為常見錯誤，且不會有任何錯誤提示。

Navigation Race Condition

Portal 每 2 秒 polling 一次 result file mtime。若模組執行後需要跳頁， 結果檔案寫入（觸發 polling）會早於 components.html() 的 postMessage 觸發。

Portal 已內建保護：EXECUTE_START 設定 10 秒 suppression； SWITCH_TAB handler 再延長 6 秒。合計保護約 16 秒不被 polling 覆蓋。 一般情況下不需要額外處理。

可用 MessageTypes

type用途payload
SWITCH_TAB跳轉到指定模組{plugin_id, tab}
EXECUTE_START通知 Portal 開始執行（框架自動送出）{}
EXECUTE_COMPLETE通知 Portal 執行完畢（框架自動送出）{success, error?}

測試規範

每個模組至少 8 個 pytest 測試，覆蓋 happy path、edge case、與錯誤處理。

{NNN}_process_test.pyimport os
import pytest
from pathlib import Path
from {NNN}_process import execute_logic

# ─── Fixtures ───────────────────────────────────────────────────────

@pytest.fixture
def tmp_db(tmp_path, monkeypatch):
    monkeypatch.setenv("CIM_LOG_DIR", str(tmp_path))
    return tmp_path

def _make_png_bytes() -> bytes:
    from PIL import Image
    import io
    img = Image.new("RGB", (4, 4), color=(128, 128, 128))
    buf = io.BytesIO()
    img.save(buf, format="PNG")
    return buf.getvalue()

# ─── Happy path ─────────────────────────────────────────────────────

def test_returns_image_b64():
    result = execute_logic({"file_bytes": _make_png_bytes(), "threshold": 100})
    assert "image_b64" in result
    assert isinstance(result["image_b64"], str)

def test_image_b64_is_valid_base64():
    import base64
    result = execute_logic({"file_bytes": _make_png_bytes(), "threshold": 100})
    decoded = base64.b64decode(result["image_b64"])
    assert len(decoded) > 0

def test_threshold_applied():
    result = execute_logic({"file_bytes": _make_png_bytes(), "threshold": 200})
    assert result.get("threshold_used") == 200

def test_default_threshold():
    result = execute_logic({"file_bytes": _make_png_bytes()})
    assert result.get("threshold_used") == 128

# ─── Edge cases ─────────────────────────────────────────────────────

def test_no_image_returns_error():
    result = execute_logic({})
    assert result.get("error") == "no_image"

def test_image_name_preserved():
    result = execute_logic({"file_bytes": _make_png_bytes(), "image_name": "test.png"})
    assert result.get("image_name") == "test.png"

def test_threshold_zero():
    result = execute_logic({"file_bytes": _make_png_bytes(), "threshold": 0})
    assert "error" not in result

def test_threshold_max():
    result = execute_logic({"file_bytes": _make_png_bytes(), "threshold": 255})
    assert "error" not in result

常用 Pattern

載入 shared 模組

避免依賴 sys.path，改用 importlib.util：

import importlib.util
from pathlib import Path

def _load_shared(name: str):
    path = Path(__file__).resolve().parent.parent / "shared" / f"{name}.py"
    spec = importlib.util.spec_from_file_location(name, path)
    mod  = importlib.util.module_from_spec(spec)
    spec.loader.exec_module(mod)
    return mod

_db_shared = _load_shared("db_utils")   # 按需載入

查詢其他模組的 DB

def _query_other_module_db() -> list[dict]:
    other_db = Path(os.environ.get("CIM_LOG_DIR", "/tmp")) / "other_records.sqlite"
    if not other_db.exists():
        return []
    with sqlite3.connect(other_db) as conn:
        conn.row_factory = sqlite3.Row
        rows = conn.execute("SELECT * FROM other_table ORDER BY id DESC").fetchall()
    return [dict(r) for r in rows]

FAQ

Q: 要修改 engine.py 嗎？

不需要。engine.py 在啟動時自動掃描 scripts/*/plugin.yaml。 只需建立正確的資料夾結構，重啟 sidecar 即可。

Q: execute_logic 跑太慢怎麼辦？

框架在按下「▶ 執行」後會顯示 spinner。長時間任務可考慮分段回傳進度， 或在 output 層使用 st.progress 配合 session_state。

Q: 為什麼 postMessage 沒有效果？

最常見的兩個原因：

  1. 缺少 "source": "cim-platform"——Portal 的 isProtocolMessage() 會靜默丟棄不含此欄位的訊息。
  2. MessageType 不在白名單——確認使用的 type 已定義在 shared-protocol/src/index.js 的 MessageTypes。

Q: 測試中如何隔離 DB？

@pytest.fixture
def tmp_db(tmp_path, monkeypatch):
    monkeypatch.setenv("CIM_LOG_DIR", str(tmp_path))
    return tmp_path

DB 路徑函式在呼叫時讀取 CIM_LOG_DIR，monkeypatch 即可生效。

Q: enabled: false 和刪除資料夾有什麼差別？

enabled: false：模組仍寫入 DB（保留 ID 記錄），但不出現在選單。適合暫時停用或廢棄標記。
刪除資料夾：模組完全消失，ID 可以被重用（謹慎使用）。
